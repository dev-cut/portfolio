# 🇰🇷 언어 규칙 (Language Rules)

## 기본 원칙

이 프로젝트의 **모든** 문서, 커뮤니케이션, 그리고 **시스템 출력물(Task, Walkthrough 등)**은 **반드시 한글**로 작성해야 합니다.

## 적용 범위

1.  **시스템 UI 및 아티팩트**:
    - `task_boundary`의 `TaskName`, `TaskSummary`, `TaskStatus`는 **무조건 한글**로 작성합니다.
    - `task.md`, `implementation_plan.md`, `walkthrough.md`의 **모든 내용(제목, 설명, 체크리스트 등)**은 한글로 작성합니다.
    - 사용자에게 보여지는 모든 메시지(`notify_user`)는 한글입니다.

2.  **룰 (Rules)**: 프로젝트의 코딩 컨벤션, 린트 규칙 등 모든 규칙 설명.
3.  **워크플로우 (Workflows)**: `.agent/workflows` 내의 모든 워크플로우 파일 내용.
4.  **메모리 (Memory)**: 사용자 요청, 작업 계획, 회고 등 모든 기록.
5.  **커밋 메시지**: 커밋 메시지도 한글로 작성.

## 예외

- 코드 내의 변수명, 함수명, 파일 경로 등 프로그래밍적으로 필요한 영문 표기는 제외합니다.
- 기술 용어 중 한글로 번역 시 의미가 모호해지는 경우 원문을 병기하거나 원문을 사용할 수 있습니다.
